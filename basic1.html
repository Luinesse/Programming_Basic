깃명령

add

git add 명령어는 사용자가 작업 디렉토리의 내용을 스테이징 영역에 추가하기 위해서 사용하는 깃명령어 중 하나입니다.
여기서 스테이징 영역이란, 작업 디렉토리와 원격 저장소인 Git 저장소 사이의 임시 저장공간과 같은 영역입니다.

commit

git commit 명령어는 git add 명령어를 통해 스테이징 영역에 있는 내용을 로컬 저장소로 업데이트하는 명령어입니다.

push

git push 명령어는 git commit 명령어를 통해 로컬 저장소에 업데이트된 내용을 원격저장소로 옮기는 명령어입니다.

pull

git pull 명령어는 원격저장소에 업로드 해놓은 내용을 작업디렉토리로 가져오는 명령어입니다. 이와 유사한 명령어로는 git clone이 있습니다.
하지만 git clone 명령어는 원격저장소에 업로드된 모든 내용을 가져오는 명령어이고,
git pull은 작업디렉토리의 내용과 원격저장소의 내용을 비교하여 최신화한 후, 작업디렉토리로 가져옵니다.

CLI

CLI는 Command-Line Inteface의 약자로 키보드로 입력받아 터미널을 통해 컴퓨터와 상호작용하는 방식을 말합니다.
과거 GUI가 사용되기 이전 MS-DOS와 같은 OS에서 CLI가 사용되었으며, 현재는 명령 프롬프트와 같은 프로그램에서 사용할 수 있습니다.

GUI

GUI는 위의 CLI와는 다르게 사용자가 컴퓨터 화면상의 그래픽 요소들을 직접 조작하는 형태입니다. Windows, Linux 등 대부분의 PC, 모바일 운영체제에서 GUI를 지원하고있습니다.

게임물리와 그래픽스

게임물리

게임을 만들때 최근에는 게임엔진에서 물리엔진을 지원해 편리하게 개발이 가능하지만, 엔진을 사용하지 않을 경우 직접 물리현상을 구현해주어야 합니다.
현실세계의 지구에서 점프를하면 지면으로 떨어지는 것처럼 중력을 구현해주어야합니다.
거기에 중력가속도를 고려하여 개발하면 게임 플레이어가 어색함이 없이 자연스럽게 플레이가 가능합니다.
또한, 원운동이나 작용 반작용 법칙 등 현실세계에서 통용되는 물리법칙들을 구현해주면 게임의 완성도가 올라간다고 생각합니다.

그래픽스

게임에서 필수적인 요소로 컴퓨터 그래픽으로 표현될 이미지가 있습니다. 게임은 결국 여러개의 프레임이 합쳐져서 움직인다고 볼 수 있습니다.
여기서, 디자이너는 개발하려는 게임의 프레임 수 에 맞추어서 이미지 스프라이트를 만들어주어야합니다.
그리고 생성된 요소들은 게임 진행 중 어떤 다른 요소와 충돌할 수 있기 때문에, 자연스러운 충돌을 구현하기 위해 디자이너는 가급적 요소의 크기에 맞는사이즈로 디자인 하여야하고 뒷 배경을 날려주는 작업을 해주어야합니다.
코딩을 하는 사람은 이미지 스프라이트에 맞춰서 적절한 충돌함수를 작성 해주는 것이 필요합니다. 또한 이미지들의 이동속도를 각각 다르게하여 현실적으로 표현하는 것도 중요하다고 생각합니다.
추가적으로, 게임을 바라보는 카메라의 위치를 다르게하여 2D에서도 쿼터뷰, 탑뷰 등으로 시각적으로 다르게 구현하는것도 필요합니다.

WOL

WOL은 Wake on Lan의 약자로 컴퓨터가 특정 네트워크 메세지를 받으면 컴퓨터 전원을 키게 되는것인데, 먼저 BIOS에서 WOL설정을 해두어야만 사용가능합니다.
또한, 특정 네트워크 메세지를 받아야하므로, 컴퓨터는 꺼져있더라도 LAN카드에는 전원이 공급되고 있어야합니다. 다만 유선에서만 가능합니다.
무선에서도 사용가능한 WOWLAN이 있지만, 설정이 힘들고 지원하는 LAN카드 또한 적습니다.

AC부팅

AC 부팅또한, 위에서 서술한 WOL과 같이 BIOS에서 컴퓨터 파워 옵션세팅을 끝내고 나면 원격으로 컴퓨터의 전원을 켜고 끄는것이 가능합니다.

스레드

스레드란, 독립적으로 실행될 수 있는 최소 크기의 프로그램 단위입니다. 모든 프로세스는 한 개 이상의 스레드를 가지며, 이러한 스레드를 두 개 이상 가지는 프로세스를 멀티 스레드 프로세스라고 합니다.
이러한 방식으로 프로세서의 성능을 더욱 높일 수 있습니다. 하지만 같은 프로세스 내에서 여러 스레드가 자원을 공유하므로, 데이터의 동기화나 교착상태가 발생할 수 있으니 고려하여 프로그래밍 해야합니다.

문맥교환

문맥교환이란, 현재 CPU를 사용중인 프로세스의 CPU 제어권이 다른 프로세스로 이양되는 과정입니다.
문맥교환은 제가 생각하기에 운영체제가 CPU자원을 사용하는 과정에서 멀티 태스킹환경에서 프로세스를 전환할때, 인터럽트가 발생할 때, 사용자모드와 커널모드를 전환할때 발생합니다.
여기서 커널모드란, 컴퓨터의 모든 자원에 접근할 수 있는 모드를 말합니다.

CI/CD

CI

CI는 Continuous Integration의 약자로 여러명이 하나의 코드에 대해서 수정을 진행해도 지속적으로 통합하면서 관리할 수 있음을 뜻합니다.
CI가 없었을 때에는 개발을 마치고 배포 후에야 코드에 오류가 있는지 없는지를 검증하고 관리가 가능했으나, 이후에는 각자의 개발자가 자신이 구현해야할 기능을 구현하고
main과 통합 후 테스트가 가능해졌습니다. 요약하자면 개발자가 구현한 코드를 기존 코드와 병합하고 병합된 코드가 올바르게 동작하는지 검증합니다. 그리고 문제가 있다면 다시 처음으로 돌아가고, 문제가 없다면 배포를 진행합니다.

CD

CD는 지속적 배포라는 의미로, CI를 통해 빌드된 코드를 테스트가능한 알파나 베타버전으로 만들고, 이 버전에서 테스트를 수행하여 문제가 발생하면 수정한 뒤 정식버전으로 배포를 진행합니다.
따라서 CD를 사용하면 소프트웨어가 항상 신뢰가능한 수준에서 배포될 수 있도록 관리가 가능해집니다.

Verse

Verse는 에픽게임즈에서 개발하여 언리얼엔진5에서 사용되고 있는 프로그래밍 언어로, Python과 비슷한 문법 구조를 사용하고있습니다.
강타입 언어이므로 개발 및 배포시 오류를 찾지 못할 가능성이 낮습니다. 그리고 언어 수준에서 동시성을 지원하므로 멀티태스킹 환경에서 시스템 수준의 스레드에 의존하지 않아도 됩니다.
여기서 강타입 언어란, 다른 자료형끼리의 변환이 금지되어있고, 변환을 위해서는 명시적으로 자료형타입을 선언해주어야합니다.

언리얼 엔진

언리얼 엔진은 유니티와 함께 대표되는 게임엔진으로 C++기반 게임엔진입니다. 블루프린트를 이용한 초급자도 편리하게 작업할 수 있다는 장점이 있습니다.
최근 언리얼 엔진5에서는 위에서 서술한 바와 같이 Verse도 내부 스크립트 언어로 사용하고 있습니다.

트윈모션

트윈모션은 3D 렌더링을 사실적으로 시각화하는 도구입니다. 게임이나 건설, 건축, 애니메이션등 다양한 분야에서 사용되고 있습니다.
비슷한 프로그램으로 루미온, 엔스케이프 등이있습니다.

포톤서버

포톤서버는 멀티 플레이어 게임 개발 프레임워크로, 자신만의 멀티플레이어 게임 백엔드를 구축하고 운영하는 프레임워크를 가질 수 있습니다.
유니티에서 사용되고 있으며, C#을 이용해 서버를 구축합니다.

TCP와 UDP

TCP

TCP는 컴퓨터와 다른 컴퓨터간의 데이터 통신을 위한 규약입니다. OSI 7계층 중 4계층인 전송 계층에서 사용되며, IP와 엮어서 TCP/IP로 불리는 경우가 많습니다.
TCP 프로토콜은 크게 세가지 흐름으로 구분됩니다. 첫번째로 연결 생성, 두번째로는 자료 전송, 마지막으로 연결 종료가 있습니다.
위에서 서술한 IP는 OSI 7계층 중 3계층에 속하며, 보내는 컴퓨터에서 받는 컴퓨터까지의 통신을 책임지는 규약입니다.

UDP

UDP는 다른 프로토콜과 마찬가지로 네트워크의 두 컴퓨터 간 데이터 전송을 위한 규약입니다. TCP에 비해 빠르지만 TCP 통신에서 사용하는 핸드셰이크를 사용하지 않으므로 안정성에서 취약합니다.
대표적으로 VoIP와 DNS 서버가 UDP를 사용하여 작동합니다.
