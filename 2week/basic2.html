Jenkins

Jenkins란 모든 언어의 조합과 소스 코드 레포지토리에 대한 지속적인 통합(CI)과 지속적인 배포(CD)환경을 구축하기 위한 도구입니다. 데브옵스의 툴체인에서 빌드, 테스트, 배포를 지원하는 도구입니다.
웹 기반의 콘솔로 다양한 인증 기반과 결합이 가능하고 권한 관리 기능을 통해 안전한 빌드, 배포 환경을 구축할 수 있습니다. 또한, 수많은 플러그인을 사용하여 자동화 할 수 있어 반복되는 작업을 줄일 수 있습니다.
빌드, 배포의 결과에 대해 통지 받을 수 있는 설정이 간편하고 다양한 채널을 통해 빠르게 피드백을 받을 수 있습니다. 그리고 오픈 소스 소프트웨어로 문서화가 이루어져 있어 빌드, 배포 이외에도 다양한 적용 사례들을 참고할 수 있습니다.
따라서 플러그인을 직접 개발하여 기능을 확장하는 것 또한 가능합니다. CI,CD를 구축하기 위한 도구이기 때문에 컴파일 오류 검출, 자동화 테스트 수행, 정적 코드 분석으로 인한 코딩 규약 준수 여부를 체크하고 성능 변화 감시, 결합 테스트 환경에 대한 배포 작업등의 역할을 수행합니다.

Kubernetes

Kubernetes는 컨테이너 오케스트레이션 도구입니다. 먼저, 컨테이너란 애플리케이션의 코드를 모든 인프라에서 실행하는 데 필요한 모든 파일 및 라이브러리와 함께 번들로 제공하는 소프트웨어 배포 프로세스입니다.
기존에는 컴퓨터에서 애플리케이션을 실행하려면 컴퓨터의 운영체제와 일치하는 버전을 설치해야 했으나, 컨테이너화를 통해 모든 유형의 디바이스 및 운영 체제에서 실행되는 단일 소프트웨어 패키지 혹은 컨테이너를 만들게 됐습니다.
컨테이너 오케스트레이션은 위에서 설명한 컨테이너들을 자동으로 관리할 수 있는 소프트웨어 기술입니다. 컨테이너 오케스트레이션 도구들을 통해 개발자는 컨테이너를 자동으로 시작, 중지 및 관리할 수 있습니다.
또한 클라우드 애플리케이션을 정확하게 조정하고 인적 오류를 방지 할 수 있습니다. 그 중 Kubernetes는 오픈 소스 컨테이너 오케스트레이터 도구입니다. 다른 도구들과 차별화된 특징으로는 먼저 전세계적 스케일의 기업들이 참여하여 클라우드 네이티브 애플리케이션 대부분이 쿠버네티스에서 잘 돌아갑니다.
배포 방식 또한 Deployment, Statefulsets, DaemonSet, Job, CronJob 등의 다양한 배포 방식을 지원합니다. Deployment는 새로운 버전의 애플리케이션을 다양한 전략으로 무중단 배포할 수 있는 방식입니다. StatefulSets는 실행 순서를 보장하고 호스트 이름과 볼륨을 일정하게 사용할 수 있는 배포 방식입니다.
DaemonSet은 로그나 모니터링 등 모든 노드에 설치가 필요한 경우에 사용되며, 배치성 작업은 Job이나 CronJob에서 지원합니다. 또 Kubernetes는 다양한 웹 애플리케이션을 하나의 로드 밸런서로 서비스하기 위해 Ingress기능을 제공합니다.
그에 더해 클라우드를 제공하는 기능과, 클러스터를 논리적으로 구분시켜주는 Namespace와 Label기능, 접근 권한을 지정하는 시스템인 RBAC 등이 있습니다. 하지만, 복잡하고 개념 이해가 어렵다는 점과 설정 파일이 많다는 등의 단점이 존재합니다.

AWS ELB

ELB란 부하를 적절하게 분배해주는 장치인 로드 밸런서를 제공해주는 시스템입니다. 이 시스템은 자동으로 로드 밸런싱을 제공하며 시스템이 서버가 죽지 않도록 알아서 관리해줍니다. 릴리즈와 배포단계에서 서버 부하 분산 및 클라이언트 요청 처리를 지원할 때 사용됩니다.
주요 특징으로는 트래픽을 분산시켜주고, 인스턴스의 상태를 자동으로 감지하여 오류가 있는 시스템을 배제합니다. 그리고 사용자 세션을 특정 인스턴스에 고정시켜주며, SSL 암호화를 지원합니다. 이때 SSL의 경유지로 ELB를 사용하게 되면 SSL처리에 따른 부하를 ELB가 수용합니다.
또한, IPv4와 IPv6를 지원하고 CloudWatch를 통해 모니털이도 가능합니다. 사용한 시간과 통과한 트래픽에 따라 종량제로 과금이 나오기에 경제성의 장점도 있습니다. 비슷한 도구로는 NGINX가 있습니다.
여기서 SSL 암호화란, 웹사이트와 브라우저 사이에 전송되는 데이터를 암호화하여 인터넷 연결을 보호하기 위한 표준 기술입니다.

ElasticSearch

ElasticSearch는 검색 및 분석 엔진으로 로그 분석, 전체 텍스트 검색, 보안 인텔리전스, 비즈니스 분석 및 운영 인텔리전스 사용 사례에 일반적으로 사용됩니다. 장점으로는 먼저 샤드를 통해 규모가 수평적으로 늘어날 수 있다는 점이 있습니다.
여기서 샤드란, 인덱스 내부의 색인된 데이터들이 하나로 뭉쳐서 존재하지 않고 물리적 공간에 여러 부분으로 나뉘어 존재하는 부분을 샤드라고 합니다. 즉, 인덱스를 샤드를 통해 늘려놨기 때문에 규모가 수평적으로 늘어납니다. 또, 레플리카를 통해 데이터의 안전성을 보장합니다.
레플리카는 샤드의 종류 중 하나로 프라이머리 샤드와 레플리카 샤드가 있는데, 프라이머리 샤드는 데이터의 원본이고 레플리카 샤드는 프라이머리 샤드의 복제본입니다. JSON 문서를 사용하기 때문에 스키마 개념이 없는 장점과, 역색인이 가능하다는 점 등이 있습니다.
하지만 실시간 처리가 안되는 점과 트랜잭션의 미지원 등이 단점으로 꼽힙니다. ELK 스택에서 Logstash로 부터 받은 데이터를 검색 및 집계하여 필요한 정보를 획득할 때 사용됩니다.

Logstash

Logstash는 플러그인 기반의 오픈소스 데이터 처리 파이프라인 도구입니다. 플러그인 기반이기에 파이프라인의 요소들은 전부 플러그인 형태로 만들어져 있고, 플러그인 개발을 위한 프레임워크를 포함하고 있고 플러그인을 관리할 수 있는 기능도 제공합니다.
이러한 플러그인들의 조합만으로 대다수의 데이터 소스에서 JSON, XML 등의 구조화된 텍스트 뿐만 아니라 다양한 형태의 데이터를 입력받아 가공한 다음에 ElasticSearch로 넘겨줄 수 있습니다.
Logstash는 자체적으로 내장되어 있는 메모리와 파일 기반의 큐를 사용하므로 처리속도와 안정성이 높습니다. 인덱싱할 도큐먼트의 수와 용량을 종합적으로 고려해 벌크 인덱싱을 수행할 뿐 아니라 파이프라인 배치 크기 조정을 통해 병목현상을 방지하고 성능 최적화를 할 수 있습니다.
거기에 더해 ElasticSearch의 장애 상황에 대응하기 위한 재시도 로직이나 오류가 발생한 도큐먼트를 따로 보관하는 데드 레터 큐를 내장하고 있어 안정성이 높은 편입니다.
로그스태시에 들어온 데이터는 입력, 필터, 출력 순의 흐름을 가집니다.

Kibana

Kibana는 ElasticSearsh에서 색인된 데이터를 검색하고 시각화하는 기능을 제공하는 무료 오픈 소스 프론트엔드 애플리케이션입니다. 색인된 데이터를 막대 차트, 원형 차트, 표, 히스토그램, 지도 등으로 시각화하여 생성 후 데이터를 분석합니다.
이러한 요소들을 결합하여 브라우저를 통해 공유하여 대규모 데이터에 대한 실시간 분석 보기를 제공해줍니다. 이 외에도 도큐먼트 검색에서 로그 분석, 보안 취약성 찾기와 같은 기능에 접근가능하고, Elastic Stack을 관리하고 어떤 사용자가 어떤 기능에 접근 할 수 있는지 제어할 수 있는 기능이 있습니다.

Vault

Vault는 ID기반의 비밀 및 암호화 관리 시스템입니다. 데브옵스 툴체인에서 운영을 지원할 때 보안 및 권한 관리 도구로써 사용됩니다. 주요 기능으로는 비밀을 영구 저장소에 쓰기전에 암호화 하므로 액세스 권한을 얻는것 만으로는 비밀에 액세스 할 수 없도록 하는 기능이 있습니다.
그리고 AWS나 SQL 데이터베이스와 같은 일부 시스템에 대해 온디맨드로 비밀을 생성할 수 있고, 데이터를 저장하지 않고 암호화 및 해독할 수 있습니다. 또 키 롤링과 침입 시 시스템 잠금을 지원하고, 비밀에 임대 기간이 있어 임대 기간이 지나게 되면 Vault는 해당 비밀을 자동으로 취소합니다.

Slack

Slack은 클라우드 컴퓨팅 기반 인스턴트 메신저 및 프로젝트 관리용 협업 도구입니다. 웹 애플리케이션이며, 일렉트론 프레임워클르 기반으로 만들어졌습니다. 데브옵스 툴체인에서 운영을 지원하는 도구입니다.
디스코드와 유사한 점이있으나 개발자와 직장인에게 포커스가 잡혀있는 업무용 도구에 가깝습니다. 각종 플러그인을 지원하여 업무에 필요한 기능을 연동해서 사용할 수 있으며, 깃허브에 새로운 push가 있을 때 알려주는 기능이나 프로젝트 관리 툴을 연동하여 실시간으로 알려주는 기능 등이 있습니다.
유사한 소프트웨어로는 Mattermost가 있습니다.

Prometheus

프로메테우스는 현재 Kubernetes 상에서 가장 많이 사용하는 오픈 소스 기반 모니터링 시스템입니다. 데브옵스의 툴체인 중 모니터링을 지원합니다.
Kubernetes 클러스터와 Docker 컨테이너들을 손쉽게 모니터링 가능하며, 간단한 텍스트 형식으로 메트릭을 쉽게 노출할 수 있습니다. 또한 key-value 형태로 레이블을 집계한 후, 대시보드 시스템에서 그래프로 쉽고 간단하게 대시보드를 만들 수 있습니다.
Docker는 애플리케이션과 의존성을 패키지화하여 실행 환경에 독립적으로 배포하는 도구로 컨테이너화 도구 중 하나입니다. 언어는 PromQL 쿼리 언어를 사용하여 간단하게 경고와 RuleSet을 정의할 수 있습니다.
프로메테우스에서 주의할 점은 가져온 메트릭은 어디까지나 근사치인 것입니다. 또한 싱글 호스트 아키텍처이기 때문에 확장이 불가능하고, 저장 용량이 부족하면 디스크 용량을 늘리는 것 밖에는 방안이 없습니다. 또한, 서버가 다운되거나 설정 변경등을 위해서 리스타트를 하게되면 그간의 메트릭은 저장되지않고 유실된다는 문제점도 가지고 있습니다.

ifconfig

ifconfig는 interface configuration의 약자로 리눅스의 네트워크 관리를 위한 인터페이스 구성 유틸리티입니다. 리눅스 명령줄 인터페이스에서는 top 명령어와 함께 가장 많이 쓰이는 명령어 중 하나 입니다. 주로 현재 네트워크 구성정보를 표시하고 네트워크 인터페이스에 IP 주소, 넷 마스크 또는 broadcast 주소를 설정하고 네트워크 인터페이스의 별칭을 만드는 등 다양한 곳에 사용됩니다.
옵션으로 -a를 주어 실행하면 모든 네트워크 장치의 주소와 시리얼 값을 모두 출력합니다. 이 외에도 인터페이스 정보만 보는 ifconfig "interface" 옵션이나 인터페이스를 비활성화 하는 ifconfig "interface" down 등의 여러가지 옵션이 존재합니다.

traceroute

traceroute는 인터넷 제어 메시지 프로토콜 패킷을 전송함으로써 작동하며, 데이터 전송과 관련된 모든 라우터는 이러한 패킷을 받습니다. 이 패킷은 전송에 사용되는 라우터가 데이터를 효과적으로 전송할 수 있는지 여부에 대한 정보를 제공해줍니다.
리눅스에서는 traceroute "사이트 주소" 형태로 사용하며 사이트 주소에 도달하기 까지 거쳐가는 IP홉을 모두 출력해줍니다. 여기서 말하는 홉이란, 컴퓨터 네트워크의 한 부분으로서 출발지와 목적지 사이에 위치한 경로 내에서 그 양 끝단을 포함한 모든 네트워크 장비 중 다음 네트워크 장비로 패킷이 이동할 때마다 발생하는 카운트를 의미합니다.

클라우드 서버

클라우드 서버란 풀링된 중앙 집중식 서버로, 네트워크를 통해 호스팅 및 제공되며 여러 사용자가 온디맨드로 액세스가 가능합니다. 기존의 물리적 서버와 완전히 동일한 기능을 수행할 수 있으며, 클라우드 컴퓨팅 환경을 통해 원격으로 서비스를 제공할 수 있습니다.
클라우드 서버는 가상화를 통해 구현되며 하이퍼바이저라는 관리 소프트웨어가 물리적 서버에 설치되어 서버를 연결하고 가상화 하면, 결합된 리소스를 추상화하고 풀링하여 가상의 서버를 생성합니다. 이러한 접근 방법을 앞서 설명한 IaaS 모델이라고 합니다.
IaaS, PaaS, SaaS의 장점과 유사하게 경제성, 편의성, 확장성, 안정성에서 장점을 가집니다.

하이퍼바이저

가상 머신 모니터라고도 불리는 하이퍼바이저는 가상 머신을 생성하고 실행하는 프로세스입니다. 메모리 및 처리와 같은 단일 호스트 컴퓨터의 리소스를 가상으로 공유하여 호스트 컴퓨터가 여러 게스트 가상 머신을 지원할 수 있도록 합니다.
일반적으로 호스트의 하드웨어에서 직접 실행되는 베어메탈과 다른 컴퓨터 프로그램처럼 운영 체제에서 소프트웨어 계층으로 실행되는 호스팅 유형으로 나뉩니다.
컨테이너와 유사하다고 볼 수 있으나 컨테이너는 단일한 유형의 운영체제에서 구동할 수 있으며, 가상 머신에서는 다양한 운영체제를 구동할 수 있도록 허용합니다.

SSH 키

SSH 키는 공개 키 암호화 방식 및 인증 확인 응답 인증을 사용하는 SSH서버에 대해 자체 식별하는 방식입니다. SSH는 비대칭 암호 방식인 Public Key 암호방식을 사용하는데 이 방식은 공개키와 개인키를 사용합니다.
공개키는 평문을 RSA 암호 알고리즘을 통해 암호화 한것이고 개인키는 암호화된 평문을 다시 복호화하여 평문을 가집니다. 여기서 RSA 암호 알고리즘은 소인수분해의 난해함에 기반하여 디자인됐고, 확장된 유클리드 호제법을 사용합니다.
키를 만드는 것은 ssh-keygen 명령어를 통해 만들고 생성된 SSH 공개키를 사용하면 됩니다.
